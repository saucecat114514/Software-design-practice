## 1. 项目一句话定义

- **课程**：高级软件设计实践（软件工程专业大三实习实训，4 周 / 32 学时 / 2 学分）。
- **主线**：**AI 赋能的全流程软件工程实践** —— 不是「用 AI 辅助旧流程」，而是「以 AI 为协作伙伴，按 AI 时代方法论走完整闭环」。

- **工作目录**：`D:\Project`（整个项目唯一根目录，所有产物都放这里）。**远程仓库**：https://github.com/saucecat114514/Software-design-practice （GitHub 用户名 saucecat114514）。
- **两阶段考核**：第 1–2 周成果 = **期中考核（需求工程）**；第 3–4 周成果 = **期末考核（软件设计与体系结构）**。

### 2.4 工具链
- **Obsidian** 知识库（Karpathy 四层架构，Markdown + 双向链接 `[[]]` + Git 版本控制）。
- **compile.js**：知识库编译验证脚本（结构/命名/链接/基线一致性四类检查；**只验证不修正**）。
- **CrewAI**：多智能体框架（每个 Agent = Role + Goal + Backstory + Tool + Task）。
- **n8n**：工作流自动化引擎（工作流1=需求开发全流程；工作流2=需求变更全流程；含人工审批 Webhook 节点）。
- **PlantUML**：纯文本 UML（用例图/活动图/时序图），可被 AI 读取与版本控制。
- **Reasonix**：AI 代码生成终端（三段式记忆：元记忆/项目记忆/会话记忆；靠分层约束 + 缓存命中率优化）。
- **语言模型（A1~A6 智能体统一）**：**DeepSeek 官方 API**，model `deepseek-v4-pro`，base `https://api.deepseek.com`。API Key 存于 `D:\Project\.env`（已 gitignore，**严禁提交 GitHub**）；CrewAI/LiteLLM 用 `deepseek/deepseek-v4-pro` 前缀、读 `DEEPSEEK_API_KEY`。
- **CodeGraph**：基于 tree-sitter 的静态分析（构建正向图谱=设计意图，逆向图谱=实际代码，自动比对查漂移）。

### 2.5 知识库目录结构（Karpathy 四层 + 不可变约束）
```
项目根/
├── raw/notes/                         # 第1层 原始笔记（涉众原话，写入即不可改）
│   └── {涉众角色}-{YYYYMMDD-HHMM}-需求记录.md
├── wiki/
│   ├── summaries/                     # 第2层 总结提炼（需求清单/UML/SRS初稿/Agent配置，带版本号）
│   │   └── {文档类型}-v{x.y}.md
│   └── baselines/                     # 第3层 基线版本（评审冻结后复制进来，不可改不可删）
│       └── BL-YYYYMMDD-NN/
│           ├── SRS-正式版.md  需求清单.md  UML模型/  溯源矩阵.md
│           ├── adr/ style/ module/ topology/ constraint/ oas/   # 设计产物分目录
│           └── change/ report/                                   # 变更三剑客 + 校验报告
├── archive/                           # 第4层 归档（五层设计资产包）
└── compile.js
```
**两条铁律**：① `raw/notes` 与 `wiki/baselines` 写入后**不得修改、不得删除**；② 需求变更只**新建**基线目录，旧基线完整保留。

### 2.7 三层约束 → 三级校验对应
架构层约束(C-ARCH) ↔ 架构级漂移校验；模块层约束(C-MOD) ↔ 模块边界漂移校验；代码层约束(C-CODE) ↔ 代码规范校验。RCR 六维：架构/模块/拓扑/契约/代码工程/需求溯源。漂移三级：严重(Blocker，必修) / 一般(Major，须改) / 提示(Minor，可评审豁免)。

## 11. 红线与高频提醒

- **契约/基线先行**：先定基线再写契约，先决策(ADR)再生成代码，禁止代码先行后补文档。
- **不可变性**：raw/notes 与 baselines 写入不可改；变更只新建基线，旧基线永久保留。
- **ID 终身唯一**：所有需求/模块/接口 ID 不复用、不删除，废弃仅标状态。
- **AI 要负面约束**：给 AI 的是「不能做什么」(C-ARCH/C-MOD/C-CODE)，否则它会把训练数据里的习惯模式（如在 OrderService 里发短信）写进来。
- **变更闭环不可缺环**：CR→CIA→约束/契约更新→代码变更→CRR→ADR-005→新基线，缺一即掉档。
- **缓存命中率**：约束提示词「固定前缀(全局+代码风格) + 变动后缀(模块+接口)」，统一措辞可把命中率从 ~62% 提到 ~76%。
- **批判性反思**：期末 ≥2000 字反思是评分核心之一，平时就记「哪个环节 AI 超预期/不如预期、换工具是否仍成立」。
