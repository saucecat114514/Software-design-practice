## 一、相对 v1 的需求变更（CR-EDU-2026-0001）

### 1. 功能需求（业务口径修改）
- **FR-REF-001 退费金额计算**：在「含赠送总课时均价 + 折价系数」基础上**叠加按消耗进度的退费折损率分档**——消耗占比 ≤30%→折损 0% / ≤60%→10% / >60%→20%（阈值配置注入）；退费额 = 基础退费 × (1 − 折损率)。（DEF-001）
- **FR-TRF-002 转班费用确认**：实际支付单价**随赠课处理模式**——作废(void)=实付/购买课时（不含赠送）；结转(carry)=实付/(购买+赠送)（含赠送，与退费均价口径对齐）。（DEF-004）

### 2. 接口需求（IFR 实现降级，契约不变）
- IFR-PAY-001 / INV-001 / MSG-001 / MINI-001：对接真实第三方 → **本地 Mock 适配**（OpenAPI 契约结构不变，标 `x-impl-mode: mock`）。

### 3. 非功能需求（NFR 资源降级）
- 性能：100 并发/秒级压测达标 → 本地单机「功能正确，不做压测」。
- 可靠性：异地灾备 RTO30min/RPO5min → 本地备份脚本说明。
- 部署：阿里云 → 本地 / Docker 单机 + H2 嵌入式。
