## 五、建模约定与决策

1. **用例粒度**：以「一项可独立达成的业务价值」为一个用例；同模块细碎 REQ 合并；公共子流程抽 `<<include>>`，可选/异常分支用 `<<extend>>`，角色范围差异用泛化（如**校长 --|> 校区主管**：校长权限是校区主管的超集）。
2. **Guard 显式化**：活动图判定一律写成带阈值/字段的完整谓词（如 `if (折扣额≤5000元 且 折扣率≥9折?)`），杜绝"及时/适当/较低"等模糊词——这是把 A2/CCB 已消除的歧义在模型层固化。
3. **裁决落地**：凡 CCB 已裁决的口径（退费单价公式、三级审批阈值、日切时点、缓冲分钟、状态机优先级等）原样进图，不在建模阶段二次改动；图尾 `legend` 标注所依据的 ISSUE，保证可审计。
4. **可渲染与可追溯**：统一 `skinparam defaultFontName "Microsoft YaHei"` 保证中文渲染；用例图保留 `' REQ-...` 行首注释做源码级追溯；全部经 `plantuml -checkonly` + 渲染双重校验。
5. **泳道**：活动图以泳道（`|角色|`）划分职责边界，在动作发生处切换，凸显跨角色协作（如顾问↔系统↔审批人↔家长）。

## 六、本轮建模的工程问题与修复（反思素材）

| 问题 | 现象 | 修复 |
|---|---|---|
| LLM 长输出被截断后**重画整图** | `.puml` 出现「残片+完整图」拼接 | `extract_puml` 取「最后一个 @startuml→其后第一个 @enduml」，兼容重画/单图 |
| 推理型模型吃光输出预算 | 大章节(教务/财务)用例图生成为空文件 | `max_tokens` 提至 8000，给推理+输出留足额度 |
| PlantUML 行尾内联注释不被支持 | `usecase ... ' REQ-001` 行尾注释报语法错 | sanitizer 把行尾 `'注释` 自动挪到上一行（行首注释，合法）；并修正 prompt |
| 全角标点当结束符 | 动作以全角 `；` 结尾导致动作未终止 | 改 prompt 强制 ASCII `;`；对历史文件做 `；→;` 修复 |
| `note across` 置于 `stop` 之后报错 | 图尾汇总注释无法渲染 | 改用 `legend right ... endlegend`；并修正 prompt |
| 泛化方向/语义错误 | 「校区主管--|>校长」「顾问--|>教师」「业务角色--|>校长」方向反/不成立 | 人工校正为 RBAC 正确方向，删除无效泛化 |
| 泳道声明但不切换 | act_04 全部动作挤在最后一条泳道 | 强化 prompt「在动作处切换泳道、用中文、文字不重复角色前缀」后重生成 |
