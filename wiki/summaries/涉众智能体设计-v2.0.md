> 项目：教育培训机构教务收费管理系统。语言模型统一 DeepSeek `deepseek-v4-pro`。

## 0. 关键定位（务必先读）

- **A1 = 与涉众对话、获取并记录需求的「采访方」智能体**，每涉众 1 个，共 5 个。**A1 不是扮演涉众。**
- **涉众那一侧**（课程顾问/教务老师/财务/校长/系统管理员）由**教师在 5000 平台（https://210.34.148.101:5000）提供的 5 个角色 agent** 扮演。
- `教育培训机构教务收费管理系统需求文档.docx` = **与教师那 5 个涉众 agent 对话后收集到的信息**（该文档开头"你扮演课程顾问"提示词是给教师 agent 用的，不是给我们 A1）。
- **对话页面**：模拟"涉众 ↔ A1 agent"，**己方（人）= 涉众（取材自教师 5000 平台 agent）、对方 = 我们的 A1 需求获取 agent**。

> v1.0（已废弃）误把 5 个 Agent 设计成"扮演涉众"，v2.0 全部改为"采访/需求获取"视角。历史见 git。

## 1. 设计原则（讲义第 3/4 节）

- **A1 是访谈设计者造出来的"会提好问题的 AI"**：Role=资深需求获取工程师，Goal=挖全挖准该涉众需求、把模糊逼成可度量，Backstory=该涉众业务域的领域认知 + 该追问哪些点。
- **四条访谈技巧**（`prompt_kit.DIALOGUE_STRATEGY`）：场景引导（从一天工作切入，不问"要什么功能"）、痛点追问、异常探测、数据边界追问（模糊词逼成具体数字）。
- **访谈纪律**（`prompt_kit.INTERVIEWER_FRAME`）：只聚焦该涉众职责范围；不臆造需求、模糊处追问；矛盾/冲突如实记录标注、留给 A2，不在采访中调解。
- **各自获取、互不沟通**：5 个 A1 各访谈一个涉众，不跨涉众；矛盾集中到 A2 分析阶段处理。
- **结构化记录**（`prompt_kit.OUTPUT_SPEC`）：REQ 编号 / 需求来源 / 涉众原话 / 需求提炼（作为[涉众]我希望…以便…）/ 优先级 / 验收要点。

## 2. 五个 A1 需求获取智能体

| 编号 | 访谈对象 | 配置文件 | Role |
|---|---|---|---|
| A1-1 | 课程顾问 | `agents/agent_course_consultant.py` | 专门访谈课程顾问、挖掘其系统需求的资深需求获取工程师 |
| A1-2 | 教务老师 | `agents/agent_academic_affairs.py` | 专门访谈教务老师…需求获取工程师 |
| A1-3 | 财务人员 | `agents/agent_finance.py` | 专门访谈财务人员…需求获取工程师 |
| A1-4 | 校长 | `agents/agent_principal.py` | 专门访谈校长…需求获取工程师 |
| A1-5 | 系统管理员 | `agents/agent_sysadmin.py` | 专门访谈系统管理员…需求获取工程师 |

每个 agent 的 **Backstory 内置该涉众业务域的"追问清单"**（如访谈课程顾问会追问：登记字段/多孩管理/手机号查重、试听预约与反馈、报名步数与满员插班、优惠审批边界、跟进提醒与转化指标、权限与响应时间），保证采访不遗漏。各 agent 的 `PROBE_TOPICS` 对应该涉众在需求文档中的全部模块（见 [[00_项目总作战计划]] §7）。

## 3. 对话 Web 页面（`agents/web/`）

- `server.py`：本地后端（零第三方依赖，标准库 http.server）。密钥仅在服务端 `.env`，**不下发前端**。接口：`/api/agents`、`/api/chat`、`/api/save`（保存对话到 `raw/notes/{涉众}-{时间戳}-需求记录.md`）。
- `index.html`：聊天界面。顶部选访谈涉众 + 开始/重新访谈 + 整理需求 + 保存到知识库；己方=涉众（右侧）、对方=A1 agent（左侧）。
- 启动：`python agents/web/server.py` → 浏览器 `http://127.0.0.1:8000`。

## 4. 验证结论

- DeepSeek 连通性：通过（`agents/test_connection.py`）。
- A1 采访行为冒烟测试：通过——给课程顾问 agent 一句涉众开场，它以采访方身份用场景引导+痛点追问回应（"从头捋一捋…第一步会做什么…有没有特别麻烦、容易出错的地方"）。
- 对话页面后端：通过——`/api/agents` 返回 5 个；`/api/chat` 让财务 agent 自动开场提问，口吻为采访方。
