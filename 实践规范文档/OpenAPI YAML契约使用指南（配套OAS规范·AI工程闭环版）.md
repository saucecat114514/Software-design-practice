# OpenAPI YAML契约使用指南（配套OAS规范·AI工程闭环版）

# 0 核心总述

OpenAPI YAML契约不是单纯的接口文档，是**可机器执行、全链路驱动、AI可控、可自动化校验的工程真值文件**。

结合整套工程规范闭环：**ASD→MDS→DTS→TLCD→OAS契约→代码生成→测试→架构校验**，所有后端代码、前端接口、Mock数据、自动化用例、架构合规校验，**必须100%以该YAML契约为唯一依据**，禁止口头约定、代码先行、文档不一致。

# 1 契约核心使用定位（能干什么）

标准化OAS YAML契约，在AI工程体系中承担4个核心唯一角色：

- **接口唯一真值**：前后端、服务间联调唯一标准，彻底消灭接口歧义、字段不一致问题

- **AI代码生成驱动源**：一键生成合规Controller、DTO/VO、请求校验、接口注释

- **自动化测试底座**：自动生成接口测试用例、参数校验用例、异常场景用例

- **架构合规校验基线**：对接CodeGraph，校验接口是否越模块、违拓扑、破架构约束

# 2 标准使用全流程（严格固定顺序）

必须遵循工程强制流程，禁止倒置步骤：

**基线定稿 → 编写OAS契约 → 评审归档 → 产物生成 → 持续校验迭代**

## 2\.1 前置准备（必须满足）

使用YAML契约前，上游基线必须全部生效：

- ASD 架构风格已定版（通信范式、层级规则明确）

- MDS 模块划分已定版（接口归属模块明确）

- DTS 依赖拓扑已定版（接口调用关系合规）

- TLCD 三层约束已定版（架构、模块、代码约束生效）

- RTM 需求溯源基线已定版（接口有明确需求依据）

## 2\.2 契约编写与归档

1. 严格遵循《OpenAPI YAML契约生成规范》编写标准化YAML文件

2. 补全溯源字段：x\-api\-id、x\-mod\-id、x\-rtm\-id、x\-tlcd\-id

3. CCB评审通过，设置为「正式生效」状态，纳入项目基线

4. 归档至固定路径：`wiki/baselines/BL-YYYYMMDD-NN/oas/`

## 2\.3 基于契约的全链路落地使用

生效后的YAML契约，可直接落地5类工程产出，全程无人工主观编写：

### 2\.3\.1 后端AI代码生成（核心用途）

AI读取标准化YAML，自动生成**完全合规**的后端代码：

- Controller接口层：请求方式、路径、参数、注解完全对齐契约

- DTO/VO结构体：字段名、类型、必填、长度、枚举、默认值1:1复刻

- 参数校验逻辑：非空、长度、正则、数值范围自动生成校验代码

- 接口注释、文档注解、返回封装逻辑统一生成

**强制规则**：AI生成代码禁止私自修改接口路径、参数结构、返回格式，所有变更必须先改YAML契约。

### 2\.3\.2 前端/客户端接口适配

基于YAML契约可自动产出：

- 前端接口请求函数（统一封装请求、参数、响应）

- 前端TS类型定义，杜绝前后端字段不一致

- 接口Mock数据，支持前端并行开发，无需后端联调

### 2\.3\.3 自动化测试落地

工具/AI自动解析YAML，生成完整测试用例：

- 正向合规参数测试用例

- 参数缺失、参数超限、格式错误异常用例

- 权限、未登录、非法访问校验用例

### 2\.3\.4 在线接口文档自动渲染

YAML可直接导入Swagger、Knife4j等工具，自动生成可视化接口文档，无需人工维护，保证**文档与代码永久一致**。

### 2\.3\.5 架构合规自动化校验

对接CodeGraph校验体系，机器自动比对：

- 接口是否超出MDS模块职责边界（防越界）

- 接口调用关系是否符合DTS拓扑白名单（防非法依赖）

- 接口通信范式是否遵循ASD架构风格（防架构漂移）

- 接口字段、约束是否符合TLCD代码层规范（防编码不规范）

# 3 日常迭代使用规则（关键红线）

## 3\.1 变更优先级规则

**契约优先、代码滞后**

所有接口变更（增删接口、改路径、改请求方式、改字段结构、调整约束）：

1. 先修改OAS YAML契约并评审迭代

2. 再同步更新AI生成代码、前端接口、测试用例

禁止直接改代码、后补契约，一经判定属于架构违规漂移。

## 3\.2 版本联动规则

OAS契约版本必须与项目基线强同步：

- 小变更（字段说明、样例优化）：OAS小版本迭代

- 结构变更（字段增删、路径/方法修改）：OAS大版本升级，同步更新MDS/DTS/TLCD对应约束

## 3\.3 接口废弃使用规则

废弃接口必须在YAML中标记 `deprecated: true`，保留文档用于溯源，禁止删除、禁止新业务调用，旧代码逐步下线。

# 4 常用落地工具链（直接适配本规范）

- **文档渲染**：Swagger UI、Knife4j、Redoc

- **代码生成**：AI工程智能体、OpenAPI Generator

- **Mock服务**：基于YAML一键启动本地Mock接口

- **自动化校验**：CodeGraph架构校验、OpenAPI规范校验器

- **前端代码生成**：自动生成TS、Axios请求封装

# 5 常见使用误区（禁止行为）

- ❌ 将YAML仅当作文档使用，不驱动代码、不做校验

- ❌ 代码写完后反向补写OpenAPI契约

- ❌ 契约字段约束不全、无溯源ID，无法机器校验

- ❌ 接口能力超出MDS模块边界、违反DTS拓扑依赖

- ❌ 代码与契约不一致，人工随意修改生成代码

# 6 极简使用口诀

**先定基线、再写契约；契约先行、代码后出；**

**一物一源、全程溯源；机器校验、杜绝漂移；**

**迭代同步、版本对齐；契约为真、全链统一。**
