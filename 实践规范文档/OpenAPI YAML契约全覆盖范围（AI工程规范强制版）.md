# 必须定义 OpenAPI YAML 契约的全覆盖范围（AI 工程规范强制版）

结合 ASD/MDS/DTS/TLCD/OAS 整套闭环规范，**所有对外暴露、跨模块交互、可被调用的结构化接口能力，必须全部定义 OpenAPI YAML 契约**，做到「凡有调用、必有契约；凡出边界、必存规范」，无契约禁止落地代码与联调。

## 一、强制必须定义契约的核心对象（100% 全覆盖）

### 1\. 所有后端 HTTP 业务接口（核心必写）

所有对外 / 对内提供业务能力的 HTTP/HTTPS 接口，统一纳入 OAS 契约管控：

- 业务查询：列表、详情、分页、筛选、导出、统计接口

- 业务操作：新增、修改、删除、批量操作、状态变更接口

- 业务特殊能力：导入、下载、审批、回调、联动业务接口

- 所有版本化接口（v1/v2），无论新旧、常用 / 冷门

### 2\. 跨模块调用内部接口（中层拓扑约束必写）

遵循 DTS 依赖拓扑规则，**所有模块间主动调用的接口必须定义契约**：

- 业务模块 ↔ 业务模块 联动接口

- 业务模块 ↔ 基础模块 调用接口

- 禁止跨模块隐性硬调用，无契约视为非法拓扑依赖

### 3\. 服务间 / 网关 / BFF 互通接口（分布式必写）

微服务、分层架构、网关代理场景全覆盖：

- 网关转发的所有后端接口

- BFF 层聚合、透传、适配接口

- 服务之间 HTTP 互通的对内接口

### 4\. 公共通用能力接口（全局统一必写）

所有全局复用、全系统依赖的基础能力接口：

- 登录、注销、权限、角色、认证授权接口

- 文件上传、下载、预览接口

- 消息通知、验证码、字典、通用配置接口

- 全局公共查询、校验接口

### 5\. 对外开放 / 第三方对接接口（对外契约必写）

所有与外部系统、第三方平台对接的接口：

- 对外开放 API 接口

- 第三方回调接收接口

- 跨系统同步、异步推送、数据互通接口

### 6\. 自动化配套接口（可校验必写）

所有需要 Mock、自动化测试、AI 生成、流水线校验的接口：

- 需要生成 Mock 数据的接口

- 需要自动化测试用例覆盖的接口

- AI 批量生成代码的所有标准化接口

- CodeGraph 架构校验、接口合规巡检的接口

## 二、不需要定义 OpenAPI 契约的范围（明确边界）

1. **纯内部私有逻辑**：模块内部私有方法、本地工具方法、无外部调用的内部逻辑（不对外、不跨模块、无网络调用）

2. **非 HTTP 协议能力**：纯 MQ 消息消费、定时任务、本地缓存逻辑、RPC 私有调用（仅 HTTP 接口走 OAS 契约）

3. **纯静态资源**：静态页面、静态文件访问、纯视图资源

## 三、硬性约束规则（红线）

1. **需求溯源绑定**：所有 OAS 接口必须对应 RTM 需求、归属 MDS 模块、符合 DTS 拓扑、遵从 TLCD 约束。

2. **零遗漏原则**：系统所有可被外部 / 跨模块调用的 HTTP 接口，**不允许存在无契约接口**。

3. **契约先行原则**：先定义 YAML 契约、评审归档，再生成代码，禁止代码先行、后补契约。

4. **废弃必标记**：下线、废弃接口仍需保留契约并标记 `deprecated: true`，禁止直接删除。
