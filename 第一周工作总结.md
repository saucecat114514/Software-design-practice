## 四、关键决策与方法

1. **A1 定位纠偏**：明确 A1 = 采访方（不是扮演涉众），与教师确认后返工——避免了整条需求获取走错方向。
2. **本机等效落地**：不依赖课程专有工具，用「DeepSeek + 自研对话平台 + Python 运行器」复刻 A1/A2 智能体协同，全程可运行、可追溯、可入库。
3. **截断根治**：用「检测截断→自动续写拼接」从根本上解决长输出被 token 上限截断，而非简单调大上限。
4. **回退闭环严格执行**：A2 发现问题 → 补访澄清 → A2 整合 → CCB 裁决，完整走通需求工程的回退环，没有把模糊/冲突带进设计阶段。

## 五、遇到的问题与解决（含反思素材）

| 问题 | 解决 |
|---|---|
| A1 智能体方向理解错（扮演涉众 vs 采访涉众） | 与教师确认后返工，并把"己方=涉众/对方=A1"固化进页面与文档 |
| 对话保存写到错误目录（路径层级算错） | 修正 `parents[4]`，并加实地校验 |
| A1 长回复被 max_tokens 截断 | 新增 `chat_complete` 自动续写，普通轮/整理需求均不再截断 |
| 关服务丢对话 | 新增 localStorage 自动保存 + 服务端恢复点 |
| 旧 server 进程占端口致新代码不生效 | 形成"改后端先杀旧 python"的操作纪律 |
| 5 份原文太大塞不进单次模型调用 | 设计"先逐角色蒸馏、再跨涉众分析"的两步法 |
